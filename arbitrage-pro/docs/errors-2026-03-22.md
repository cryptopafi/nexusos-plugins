# Errors & Fixes — GraphQL E2E Session 2026-03-22

## Error Registry

### E1: GraphQL Schema Mismatch (CRITICAL)
**What**: Original script used wrong query (`lot(id:)`) and wrong field names (`title.value`, `currentBid.amount.value`, `numberOfBids`, `reservePriceStatus`).
**Root cause**: Schema was guessed from browser DevTools, not introspected. API evolved since initial discovery.
**Fix**: Full introspection via `__schema` + `__type` queries. Correct query = `lotDetails(displayId, locale, platform)`, Money = `cents` field, title = plain String.
**Safeguard**: Script header documents schema version date. Any future "lot not found" error → re-introspect schema first.
**Files fixed**: `scripts/troostwijk-graphql.sh`

### E2: VPS Deploy Path Wrong (HIGH)
**What**: `channel-config.yaml` had `vps_report_path: /var/www/html/nexus` but nginx serves from `/var/www/nexus/`.
**Root cause**: Path was set during initial build without verifying nginx config.
**Fix**: Updated to `/var/www/nexus`.
**Safeguard**: Reporter skill now mandates HTTP 200 check after deploy. Added to Common Errors section.
**Files fixed**: `resources/channel-config.yaml`, `skills/reporter/SKILL.md`

### E3: Search Mode No Error Handling (HIGH)
**What**: Search and list modes piped curl directly to jq without HTTP status check. API errors produced silent jq parse failures.
**Root cause**: Only single-lot mode had the full retry+error pattern. Search/list were added later without the same rigor.
**Fix**: Both modes now capture HTTP status, check for 200, emit JSON error to stderr on failure. Search continues on page failure.
**Safeguard**: Script pattern established — all curl calls must use `-w "\n%{http_code}"` pattern.
**Files fixed**: `scripts/troostwijk-graphql.sh`

### E4: GraphQL pageSize > 100 Returns Empty Silently (MEDIUM)
**What**: `pageSize: 200` returned `results: []` with no error. Wasted multiple debug cycles.
**Root cause**: API enforces max 100 per page but returns empty instead of error.
**Fix**: Script hardcodes pageSize=100 in search mode. Documented in scout-source SKILL.md.
**Safeguard**: Added to GraphQL API constraints section in scout-source. Search mode uses pagination instead.
**Files fixed**: `scripts/troostwijk-graphql.sh`, `skills/scout-source/SKILL.md`

### E5: GraphQL pageNumber Starts at 1 (MEDIUM)
**What**: `pageNumber: 0` returned validation error. API is 1-indexed.
**Root cause**: Assumed 0-indexed pagination (common in most APIs).
**Fix**: Script uses pageNumber=1 as default.
**Safeguard**: Documented in scout-source SKILL.md constraints section.
**Files fixed**: `scripts/troostwijk-graphql.sh`, `skills/scout-source/SKILL.md`

### E6: Platform Enum Not "TROOSTWIJK" (MEDIUM)
**What**: Used `platform: TROOSTWIJK` but enum is `TWK`.
**Root cause**: Guessed the enum value instead of introspecting.
**Fix**: Introspected all 13 Platform enum values. Script header documents all values.
**Safeguard**: Documented in script header and scout-source SKILL.md.
**Files fixed**: `scripts/troostwijk-graphql.sh`, `skills/scout-source/SKILL.md`

### E7: Title Contains \r Carriage Return (LOW)
**What**: Some lot titles from GraphQL contain `\r` characters (Windows line endings in their DB).
**Root cause**: Troostwijk data entry with Windows clients.
**Fix**: All 3 script modes now strip `\r` via jq `gsub("\\r"; "")`.
**Files fixed**: `scripts/troostwijk-graphql.sh`

### E8: Scout-Source Field Names Stale (MEDIUM)
**What**: scout-source SKILL.md documented old field names (`closing_datetime`, `reserve_status`, `markup_percentage`) that don't match updated script output (`closing_timestamp`, `sale_term`, `buyer_premium_pct`).
**Root cause**: SKILL.md was written for v1 script, not updated when script was rewritten.
**Fix**: Updated all field names and added complete field reference with types.
**Safeguard**: Field reference now includes source (lotDetails vs allLots) and data type.
**Files fixed**: `skills/scout-source/SKILL.md`

### E9: optimization-proposals.json Missing (LOW)
**What**: Step 10.5 (Karpathy Loop) references `state/optimization-proposals.json` but file didn't exist.
**Root cause**: State file mentioned in procedure but never seeded.

**Files fixed**: `state/optimization-proposals.json`

### E10: ARBITRAGE-SOC Wrong Path Reference (LOW)
**What**: Procedure referenced `resources/category-intelligence.yaml` but file is at `state/category-intelligence.yaml`.

**Files fixed**: `procedures/ARBITRAGE-SOC.md`

## Prevention Checklist (for future sessions)

1. **GraphQL changes**: If any query fails with "Property X does not exist" → introspect `__type` before guessing
2. **VPS deploy**: ALWAYS verify HTTP 200 after SCP. If 404 → check nginx alias path.
3. **New script modes**: Copy the error handling pattern from single-lot mode (HTTP check + backoff)
4. **API pagination**: Test with small pageSize first. Never assume 0-indexed.
5. **Field name changes**: When rewriting a script, grep all SKILL.md files for old field names
6. **State files**: When referencing a state file in a procedure, verify it exists or create it
7. **Path references**: `resources/` = immutable config. `state/` = mutable runtime data. Don't mix.
